# v0.6.0 Release Notes

## 릴리스 성격

`v0.6.0`은 실제 운영 배포가 아니라, 핀테크 학습 프로젝트의 첫 검증 기준선이다.

이 릴리스는 모놀리식 Spring Boot 애플리케이션 안에서 돈 이동, 정합성 기록, PostgreSQL 영속화, outbox relay 운영 흐름을 문서와 테스트로 설명할 수 있는 상태를 고정한다.

## 포함 기능

- 잔액 조회와 거래내역 조회 API
- 회원/지갑 계정 조회 API
- 충전/송금 1차 흐름
- 멱등키 기반 중복 요청 수렴
- 원장과 감사 로그 기록
- PostgreSQL 저장소 프로필과 Testcontainers 검증
- Flyway 기반 스키마 마이그레이션
- PostgreSQL row lock과 lock timeout 정책
- 충전/송금 논리적 트랜잭션 단계 로그
- Transactional Outbox 적재
- Outbox relay 상태, claim, retry, lease recovery
- Max attempt 이후 manual review 격리
- Manual review 조회와 requeue API
- Requeue operator/reason 감사 이력

## 변경된 도메인 규칙

- 충전/송금 성공 결과는 잔액, 거래내역, 원장, 감사 로그, step log, outbox event로 추적한다.
- outbox event는 `PENDING`, `PROCESSING`, `PUBLISHED`, `FAILED`, `MANUAL_REVIEW` 상태를 가진다.
- 자동 발행 재시도는 최대 3회로 제한하고, 초과 시 manual review로 격리한다.
- manual review event를 requeue할 때 `operator`, `reason`을 필수로 받고 별도 감사 이력에 저장한다.

## 검증 기준

릴리스 PR에서 다음 명령을 통과해야 한다.

```bash
./gradlew check
docker compose config
git diff --check
```

GitHub Actions의 `Gradle Check`가 통과한 뒤 `v0.6.0` tag와 GitHub Release를 발행한다.

## 알려진 리스크

- 실제 외부 broker 발행은 아직 없다.
- 인증/인가가 없으므로 관리자 API를 운영 환경에 노출할 수 없다.
- requeue `operator`는 아직 인증 주체와 연결되지 않는다.
- 승인 워크플로우, 알림, 모니터링 연동은 없다.
- 실제 금융 거래, 실명 인증, 외부 결제망 연동, 개인정보 처리는 범위 밖이다.
