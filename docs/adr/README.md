# Architecture Decision Records

이 폴더는 `ai-repo`의 중요한 기술, 운영, 도메인 결정을 기록합니다.

ADR은 이 저장소에서 중요한 결정의 source of truth입니다. README와 Wiki는 ADR을 요약하거나 연결할 수 있지만, 결정 자체를 대체하지 않습니다.

## 작성 원칙

- 결정이 필요한 배경과 제약을 먼저 설명합니다.
- 최소 2개 이상의 선택지를 비교합니다.
- 선택한 결정뿐 아니라 포기한 선택지의 비용도 기록합니다.
- 결정의 결과와 후속 작업을 명시합니다.
- 결정이 바뀌면 기존 ADR을 수정하기보다 새 ADR로 대체 관계를 남깁니다.
