# [Feature] 원장/감사 로그 1차 모델과 조회 API 구현

## 작업 요약

충전/송금으로 발생한 잔액 변경을 거래내역과 분리된 원장 기록으로 남기고, 명령 처리 결과를 감사 로그로 추적한다. 초기 범위는 인메모리 저장소와 조회 API로 제한한다.

## 배경과 문제

충전/송금 1차 구현은 잔액 변경과 거래내역 기록을 제공한다. 그러나 핀테크 서비스에서는 사용자에게 보여주는 거래내역과 시스템 정합성 검증을 위한 원장/감사 기록을 분리해야 한다.

현재 남아 있는 문제는 다음과 같다.

- 잔액 변경의 결과 잔액을 검증할 독립 기록이 없다.
- 충전/송금 명령이 어떤 요청으로 처리되었는지 감사할 수 없다.
- 거래내역이 사용자 조회와 정합성 검증 책임을 동시에 가진다.
- 이후 PostgreSQL 영속화와 원장 불변성 제약을 설계할 기준 모델이 없다.

## 작업 유형

도메인 기능

## 도메인 영역

원장/감사 로그

## 범위

### 하는 것

- 원장 엔트리 도메인 모델을 추가한다.
- 감사 이벤트 도메인 모델을 추가한다.
- 충전/송금 성공 시 원장 엔트리를 기록한다.
- 충전/송금 성공 시 감사 이벤트를 기록한다.
- 원장 조회 API를 추가한다.
- 감사 이벤트 조회 API를 추가한다.
- 단위 테스트와 API 통합 테스트를 추가한다.
- ADR과 Wiki에 원장/거래내역/감사 로그의 책임 경계를 기록한다.

### 하지 않는 것

- PostgreSQL 영속화는 이번 단계에서 구현하지 않는다.
- 원장 수정 불가를 DB 제약으로 보장하지 않는다.
- 회계식 복식부기 전체 모델은 구현하지 않는다.
- 관리자 인증/인가와 감사 로그 접근 제어는 구현하지 않는다.
- 장애 보상, 취소, 실패 원장 기록은 후속 단계로 미룬다.

## 수용 기준

- [ ] 충전 성공 시 원장 엔트리가 생성된다.
- [ ] 송금 성공 시 출금 지갑과 입금 지갑 각각에 원장 엔트리가 생성된다.
- [ ] 원장 엔트리는 잔액 변경 후 잔액을 가진다.
- [ ] 같은 멱등키 재시도는 원장 엔트리를 중복 생성하지 않는다.
- [ ] 충전/송금 성공 시 감사 이벤트가 생성된다.
- [ ] `GET /api/v1/wallets/{walletId}/ledger-entries`로 원장 엔트리를 조회할 수 있다.
- [ ] `GET /api/v1/audit-events`로 감사 이벤트를 조회할 수 있다.
- [ ] 문서와 ADR에 거래내역, 원장, 감사 로그의 차이가 명시되어 있다.

## 도메인 규칙과 불변식

- 원장 엔트리는 잔액 변경의 검증 기록이다.
- 거래내역은 사용자 조회용 기록이다.
- 감사 로그는 명령 처리와 시스템 행위의 추적 기록이다.
- 원장 엔트리는 생성 후 수정하지 않는 것을 목표로 한다.
- 원장 엔트리는 변경 후 잔액을 반드시 가진다.
- 성공한 충전/송금은 거래내역, 원장, 감사 로그를 함께 남긴다.

## 대안과 트레이드오프

### 대안 A: 거래내역을 원장으로 계속 사용

장점:

- 모델과 저장소 변경이 가장 작다.
- 현재 API를 크게 바꾸지 않아도 된다.

단점:

- 사용자 표시 요구와 정합성 검증 요구가 섞인다.
- 감사/회계 요구가 늘어날수록 거래내역 모델이 비대해진다.
- 원장 불변성, 보정, 재처리 설계를 분리하기 어렵다.

### 대안 B: 원장과 감사 로그를 1차 모델로 분리

장점:

- 잔액 변경 검증 기록과 시스템 처리 기록이 명확히 분리된다.
- PostgreSQL 영속화와 불변성 제약 설계의 기준점이 생긴다.
- 충전/송금 멱등성 검증을 더 강하게 테스트할 수 있다.

단점:

- 인메모리 저장소 책임이 커진다.
- 아직 DB 제약이 없어 불변성을 프로세스 수준 규칙으로만 표현한다.

### 현재 선호안

원장과 감사 로그를 1차 도메인 모델로 분리하되, 저장소는 인메모리로 유지한다. PostgreSQL과 DB 제약은 다음 단계에서 도입한다.

## 릴리스 고려사항

- 실행 검증: `./gradlew check`와 로컬 원장/감사 조회 API 확인.
- 마이그레이션: 아직 인메모리라 없음.
- 알려진 리스크: 프로세스 재시작 시 원장/감사 로그가 초기화된다.
