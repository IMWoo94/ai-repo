# [Docs] 백엔드/프론트 테스트 가이드 정리

GitHub Issue: https://github.com/IMWoo94/ai-repo/issues/49

## 배경

백엔드 테스트, 프론트 build, 프론트 E2E가 모두 생겼지만 로컬에서 어떤 순서로 무엇을 실행해야 하는지 한 곳에서 보기 어렵다. IntelliJ와 터미널에서 재현 가능한 테스트 가이드가 필요하다.

## 목표

- 백엔드/프론트/E2E 테스트 실행 가이드를 하나의 문서로 정리한다.
- 로컬 E2E 시연 결과와 기대 출력 기준을 문서화한다.
- CI job과 로컬 명령의 대응 관계를 명확히 한다.
- 실패 시 1차 확인 포인트를 정리한다.

## 범위

- 로컬 테스트 통합 가이드 문서
- README 테스트 섹션 연결
- frontend/E2E testing 문서 연결
- progress report
- issue draft

## 범위 제외

- 테스트 코드 추가
- CI workflow 변경
- 신규 E2E 시나리오 추가

## 수용 기준

- [ ] 백엔드 테스트 명령과 목적이 문서화되어 있다.
- [ ] 프론트 build/E2E 명령과 목적이 문서화되어 있다.
- [ ] Playwright E2E가 서버를 자동 실행한다는 점이 문서화되어 있다.
- [ ] 로컬 시연 통과 기준이 문서화되어 있다.
- [ ] README 또는 기존 testing 문서에서 새 가이드로 연결된다.
